# Node Express Template

A Deno + Express boilerplate for building REST APIs with PostgreSQL, Supabase authentication, and Kysely ORM. Clone this repository to get a production-ready starting point for any new project.

## Tech Stack

| Layer | Technology |
|---|---|
| Runtime | [Deno](https://deno.com) |
| Framework | [Express 5](https://expressjs.com) |
| Database | PostgreSQL via [Kysely](https://kysely.dev) |
| Auth | [Supabase](https://supabase.com) |
| Validation | [Zod](https://zod.dev) |
| API Docs | [swagger-jsdoc](https://github.com/Surnet/swagger-jsdoc) + [swagger-ui-express](https://github.com/scottie1984/swagger-ui-express) |
| Logging | [Morgan](https://github.com/expressjs/morgan) |
| Security | [Helmet](https://helmetjs.github.io), [express-rate-limit](https://github.com/express-rate-limit/express-rate-limit) |

## Getting Started

### 1. Prerequisites

- [Deno](https://deno.com/manual/getting_started/installation) >= 2.x
- A running PostgreSQL database
- A [Supabase](https://supabase.com) project (for authentication)

### 2. Environment Setup

Copy `.env.template` to `.env` and fill in your values:

```sh
cp .env.template .env
```

```env
PORT=5000
DATABASE_URL=postgres://user:password@host:5432/database
SUPABASE_KEY=your-supabase-anon-key
SUPABASE_URL=https://your-project.supabase.co
```

### 3. Install Dependencies

```sh
deno install
```

### 4. Run Migrations

```sh
deno task kysely migrate:latest
```

### 5. Generate Database Types

After running migrations, generate TypeScript types from your schema:

```sh
deno task gen-types
```

This overwrites `src/database/types.ts` with a fully typed `DB` interface. Do not edit this file manually.

### 6. Start the Development Server

```sh
deno task dev
```

The server starts on the configured `PORT` (default: `5000`) with file watching enabled.

On every startup, Swagger docs are regenerated and served at `http://localhost:<PORT>/docs`.

## API Routes

Routes are auto-discovered. Any directory under `src/api/` that contains an `index.ts` is registered automatically. Directories prefixed with `_` are skipped (use them for shared utilities).

### Health Check

```
GET /api/v1
→ { "status": "ok", "version": "v1" }
```

## Adding a New Route

Use the scaffold script to generate a new route module:

```sh
deno task gen-route --name users
# Creates: src/api/v1/users/index.ts
#          src/api/v1/users/controllers.ts
#          src/api/v1/users/services.ts
```

To target a different API version:

```sh
deno task gen-route --name posts --version 2
# Creates: src/api/v2/posts/...
```

The generated files follow this pattern:

**`index.ts`** — Router with route definitions, validation, and OpenAPI docs

```typescript
import { Router } from 'express';
import { z } from 'zod';
import { myController } from './controllers.ts';
import { auth } from '../../../middlewares/auth.ts';
import validate from '../../../middlewares/validate.ts';

const router = Router();

const createSchema = z.object({
    name: z.string().min(1),
});

/**
 * @openapi
 * /v1/users:
 *   get:
 *     tags: [users]
 *     summary: Get all users
 *     security:
 *       - bearerAuth: []
 *     responses:
 *       200:
 *         description: Success
 *       401:
 *         description: Unauthorized
 *   post:
 *     tags: [users]
 *     summary: Create a user
 *     security:
 *       - bearerAuth: []
 *     requestBody:
 *       required: true
 *       content:
 *         application/json:
 *           schema:
 *             type: object
 *             required: [name]
 *             properties:
 *               name:
 *                 type: string
 *     responses:
 *       200:
 *         description: Created
 *       400:
 *         description: Validation error
 *       401:
 *         description: Unauthorized
 */
router.get('/', auth, myController);
router.post('/', auth, validate({ body: createSchema }), myController);

export default router;
```

**`controllers.ts`** — Request handlers

```typescript
import { Controller } from '../../../types.d.ts';
import { myService } from './services.ts';
import { createResponse } from '../../../utils/response.ts';

export const myController: Controller = async (req, res, next) => {
    const data = await myService(req.user!.id);
    return createResponse(res, { data, message: 'Success', status: 200 });
};
```

**`services.ts`** — Business logic and database queries

```typescript
import { db } from '../../../database/index.ts';

export const myService = async (userId: string) => {
    return await db.selectFrom('users').where('id', '=', userId).selectAll().execute();
};
```

## Validation

Use the `validate` middleware to validate request inputs with Zod before they reach the controller. It accepts an object with optional `body`, `query`, and `params` schemas.

```typescript
import validate from '../../../middlewares/validate.ts';
import { z } from 'zod';

const bodySchema = z.object({
    name: z.string().min(2).max(50),
    email: z.string().email(),
});

const querySchema = z.object({
    page: z.coerce.number().int().min(1).optional(),
});

const paramsSchema = z.object({
    id: z.string().uuid(),
});

router.post('/', validate({ body: bodySchema }), myController);
router.get('/:id', validate({ params: paramsSchema, query: querySchema }), myController);
```

When validation fails, a `400` response is returned automatically with the first failing field's message. No try/catch needed — Express 5 catches the thrown `ZodError` and routes it to the global error handler.

## API Documentation (Swagger)

Swagger docs are **auto-generated on every server start** by scanning all `src/api/**/*.ts` files for `@openapi` JSDoc comments. No separate build step required.

### Accessing the Docs

```
http://localhost:<PORT>/docs
```

### Writing OpenAPI Docs

Add a `@openapi` JSDoc block directly above or near your route definition in `index.ts`:

```typescript
/**
 * @openapi
 * /v1/users/{id}:
 *   get:
 *     tags: [users]
 *     summary: Get a user by ID
 *     security:
 *       - bearerAuth: []
 *     parameters:
 *       - in: path
 *         name: id
 *         required: true
 *         schema:
 *           type: string
 *           format: uuid
 *     responses:
 *       200:
 *         description: User found
 *         content:
 *           application/json:
 *             schema:
 *               type: object
 *               properties:
 *                 id:
 *                   type: string
 *                 name:
 *                   type: string
 *       404:
 *         description: Not found
 *       401:
 *         description: Unauthorized
 */
router.get('/:id', auth, validate({ params: paramsSchema }), myController);
```

### Authentication in Swagger UI

Protected endpoints use `bearerAuth`. In the Swagger UI, click **Authorize** and paste your access token (without the `Bearer` prefix). Get a token with:

```sh
deno task get-token --email test@example.com --password secret123
```

### Customizing the Spec Info

Update the `swaggerOptions.definition` in `src/utils/swagger.ts` to set the API title, version, and description for your project.

## Database

### Creating a Migration

Add a new file to `src/database/migrations/` using the timestamp naming convention:

```
[unix_timestamp_ms]_[description].ts
```

Each migration exports `up` and `down` functions:

```typescript
import { Kysely, sql } from 'kysely';

export async function up(db: Kysely<never>): Promise<void> {
    await db.schema
        .createTable('posts')
        .addColumn('id', 'uuid', (col) => col.primaryKey())
        .addColumn('title', 'varchar(255)', (col) => col.notNull())
        .addColumn('created_at', 'timestamp', (col) =>
            col.notNull().defaultTo(sql`now()`))
        .execute();
}

export async function down(db: Kysely<never>): Promise<void> {
    await db.schema.dropTable('posts').execute();
}
```

After adding a migration, run:

```sh
deno task kysely migrate:latest
deno task gen-types
```

### Creating a Seed

Add a file to `src/database/seeds/` exporting a `seed` function:

```typescript
import { Kysely } from 'kysely';
import { DB } from '../types.ts';

export async function seed(db: Kysely<DB>): Promise<void> {
    await db.insertInto('users').values({ ... }).execute();
}
```

## Authentication

The `auth` middleware (`src/middlewares/auth.ts`) validates a Supabase JWT and attaches the authenticated user to `req.user`.

Apply it per-route:

```typescript
import { auth } from '../../../middlewares/auth.ts';

router.get('/me', auth, myController);
```

Inside the controller, the user is available as `req.user` (typed as `SupabaseUser`).

To extend the user with a database profile, query your `users` table inside the middleware or in the controller after authentication.

### Dev Utilities

Register a test user:

```sh
deno task add-user --email test@example.com --password secret123
```

Get an access token for testing:

```sh
deno task get-token --email test@example.com --password secret123
```

Use the token in requests:

```
Authorization: Bearer <token>
```

## Error Handling

Throw `ApiError` from anywhere in a controller or service to return a structured error response:

```typescript
import ApiError from '../../../utils/ApiError.ts';

throw new ApiError(404, 'Resource not found');
```

Zod validation errors are caught automatically and return a `400` with the first failing message.

All unhandled errors fall through to `src/middlewares/errorHandler.ts`, which returns a `500` response.

## Response Format

Use `createResponse` for all responses to maintain a consistent shape:

```typescript
import { createResponse } from '../../../utils/response.ts';

createResponse(res, {
    status: 200,
    message: 'Success',
    data: { ... },
});
```

Response shape:

```json
{
  "status": 200,
  "message": "Success",
  "data": {}
}
```

## Coding Conventions

- **Separation of concerns**: routes in `index.ts`, request handling in `controllers.ts`, business/DB logic in `services.ts`.
- **No inline logic in routes**: route files only wire up controllers, middleware, and OpenAPI docs.
- **Controllers never query the database directly**: all DB access goes through services.
- **Validation at the boundary**: use `validate()` in the route before the controller, keep schemas in `index.ts`.
- **Naming**: functions and variables use `camelCase`, files use `kebab-case`.
- **Formatting**: enforced by `deno fmt` (4-space indent, single quotes, semicolons, 80-char line width).
